# 🎯 Crosswalk Classifier - PyTorch Binary Image Classification

A complete Deep Learning project for detecting crosswalks (Fussgängerstreifen) in 25m x 25m satellite images from Swisstopo using PyTorch and Transfer Learning.

## 📋 Project Overview

**Goal**: Build a production-ready binary classifier that detects whether a satellite image contains a crosswalk or not.

**Architecture**: 
- **Backbone**: ResNet18 (pre-trained on ImageNet) - frozen for feature extraction
- **Head**: 3 custom linear layers with batch normalization and dropout for binary classification
- **Optimization**: Optuna-based hyperparameter tuning

**Key Features**:
- ✅ Data augmentation optimized for satellite imagery (flips, rotations, color jitter)
- ✅ Reproducible results with strict seed setting
- ✅ GPU-optimized training with early stopping
- ✅ Comprehensive evaluation with confusion matrix and ROC curves
- ✅ Automated hyperparameter optimization with Optuna



- Train ResNet18 with custom classification head

- Save best model checkpoint
- Save training history

## 📊 Model Architecture Details

### Backbone: ResNet18 (Pre-trained)
```
Input (3, 224, 224)
    ↓
ResNet18 Backbone (Frozen)
    ├─ Conv1 (3 → 64 channels)
    ├─ Layer1-4 (residual blocks)
    └─ Global Average Pooling → (512,)
    ↓
Custom Classification Head
    ├─ FC1: 512 → 256
    ├─ BatchNorm1d → ReLU → Dropout(0.5)
    ├─ FC2: 256 → 128
    ├─ BatchNorm1d → ReLU → Dropout(0.5)
    ├─ FC3: 128 → 2 (logits)
    ↓
Output: [logits for "crosswalk", "no-crosswalk"]
```

### Why This Architecture?

1. **Transfer Learning**: ResNet18 learns generic image features from ImageNet
2. **Frozen Backbone**: Prevents overfitting on small datasets
3. **3-Layer Head**: Sufficient capacity for binary classification without overfitting
4. **Batch Normalization**: Stabilizes training and reduces internal covariate shift
5. **Dropout**: Regularization to prevent overfitting
6. **Adam Optimizer**: Adaptive learning rates, robust to hyperparameter choices

## 🎨 Data Augmentation

Applied only during **training**:
- **Random Horizontal Flip** (p=0.5)
- **Random Vertical Flip** (p=0.5)
- **Random Rotation** (0-90°) - Satellite images are rotation-invariant
- **Color Jitter** (brightness ±20%, contrast ±20%, saturation ±20%, hue ±10%)
- **Normalization**: ImageNet statistics (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])

Why these augmentations?
- Crosswalks appear in any orientation
- Different seasons/lighting conditions affect colors
- Prevents model from memorizing dataset

---

## 📈 Training Strategy

1. **Optimizer**: Adam (adaptive learning rates)
2. **Loss Function**: CrossEntropyLoss (balanced for two classes)
3. **Learning Rate Scheduler**: ReduceLROnPlateau
   - Reduces LR by 50% if validation accuracy plateaus for 3 epochs
4. **Early Stopping**: Stop if validation accuracy doesn't improve for 5 epochs
5. **Gradient Clipping**: max_norm=1.0 to prevent exploding gradients
6. **Weight Decay**: L2 regularization (1e-5) to prevent overfitting

## 💾 Reproducibility

All scripts set random seeds for deterministic behavior:

```python
def set_seed(seed: int = 42):
    torch.manual_seed(seed)
    torch.cuda.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
    torch.backends.cudnn.deterministic = True
    torch.backends.cudnn.benchmark = False
```

This ensures:
- Same training results with same code + data
- Fair comparison between model versions
- Reproducible evaluation for hackathon grading

## 🔬 Hyperparameter Tuning with Optuna

### Why Optuna?
- **Efficient**: Uses Bayesian optimization (TPE sampler)
- **Parallel**: Can run multiple trials simultaneously
- **Pruning**: Automatically stops unpromising trials early
- **Visualization**: Built-in plotting and analysis tools

### Recommended Workflow:
1. Train baseline model with default hyperparameters → `04_train.py`

## 📊 Evaluation Metrics

The evaluation script (`05_evaluate.py`) computes:

1. **Overall Metrics**:
   - Accuracy: (TP + TN) / (TP + TN + FP + FN)
   - Precision: TP / (TP + FP) - of detected crosswalks, how many are correct?
   - Recall: TP / (TP + FN) - of all crosswalks, how many did we find?
   - F1-Score: Harmonic mean of precision and recall
   - ROC-AUC: Area under ROC curve (0.5=random, 1.0=perfect)

2. **Confusion Matrix**:
   - True Positives (TP): Correctly predicted crosswalk
   - True Negatives (TN): Correctly predicted no-crosswalk
   - False Positives (FP): Incorrectly predicted crosswalk
   - False Negatives (FN): Incorrectly predicted no-crosswalk

3. **Per-Class Metrics**: Accuracy for each class separately

## 🐛 Troubleshooting

### GPU Not Available
```python
# Check in terminal
python -c "import torch; print(torch.cuda.is_available())"
```
If False: Install CUDA-compatible PyTorch from pytorch.org

### Out of Memory (OOM)
- Reduce batch size in training config (32 → 16)
- Reduce model complexity if needed
- Clear GPU cache: `torch.cuda.empty_cache()`

### No Improvement During Training
- Increase learning rate (1e-3 → 1e-2)
- Increase dropout rate to prevent overfitting
- Check data augmentation is working

### Data Not Found
- Ensure `data/asconalocarno/y/` and `data/asconalocarno/n/` exist
- Run `01_data_split.py` to create train/test structure
- Check image file extensions are supported
